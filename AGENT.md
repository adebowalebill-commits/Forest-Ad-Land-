# AGENT.md — Forest Ad Land

## Project Identity

**Forest Ad Land** is a Web3 ecosystem combining digital land ownership, advertising spaces, a utility token, a marketplace, and community-driven growth. The landing page is the public-facing introduction to this project.

---

## Agent Objective

Build a **premium, informational landing page** that clearly communicates the Forest Ad Land vision, builds community trust, and directs visitors to join the ecosystem. The page must feel serious, transparent, and long-term — not like short-term hype.

---

## Core Rules

### Do
- Keep the design **premium, modern, and Web3-native** (dark mode, glassmorphism, green accents, ambient motion).
- Use **vanilla HTML, CSS, and JavaScript** — no frameworks, no build tools.
- Make the site **fully responsive** across all devices.
- Write **semantic HTML** with proper heading hierarchy and accessibility.
- Maintain a **transparent, honest tone** in all copy — no fake hype, no unrealistic promises.
- Include the **contract address (CA)** with a copy-to-clipboard feature.
- Link to all official socials: Telegram, X (Twitter), Email.
- Keep **performance** high — the page should load fast with no unnecessary overhead.
- Use **scroll-triggered animations** and hover effects to make the page feel alive.
- Preserve all existing comments and documentation unless explicitly told to modify them.

### Don't
- Do NOT build a dApp, dashboard, or interactive Web3 product — this is informational only.
- Do NOT use any JavaScript frameworks (React, Vue, etc.) or CSS frameworks (Tailwind, Bootstrap).
- Do NOT use placeholder images — generate real assets with the image generation tool.
- Do NOT write copy that sounds like a scam or pump-and-dump (no "guaranteed returns," "moon," "100x," etc.).
- Do NOT rush exchange/listing language — the project philosophy is to build substance first.
- Do NOT make the site overly complex for regular (non-crypto) users to understand.
- Do NOT add wallet-connect, swap, or staking features — those come later.

---

## Tone & Voice

- **Confident but grounded** — this is a real project with a real vision.
- **Clear and accessible** — avoid excessive crypto jargon; anyone should understand the basics.
- **Community-first** — emphasize that early supporters matter and help build the project.
- **Transparent** — the roadmap is public, progress is real, no hidden agendas.
